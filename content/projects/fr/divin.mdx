## Aperçu

**Div-In** est une application mobile communautaire dédiée au monde de la plongée.  
Elle permet aux utilisateurs de gérer leurs sorties, de suivre leur progression, de partager des informations et, à terme, de connecter plongeurs et structures.

Je participe au projet en tant que **développeur full-stack en appui**, principalement sur :

- l’implémentation de **petites features ciblées** côté **React Native** et côté **backend Laravel**,
- la **correction de bugs** et l’amélioration de l’expérience utilisateur,
- la **rédaction et la structuration de documentation technique** pour clarifier le fonctionnement de l’app.

---

## Fonctionnalités sur lesquelles j’ai contribué

Sans être développeur principal du projet, j’ai apporté des contributions concrètes sur plusieurs axes :

- **Écrans & flux React Native**
  - Ajustements sur des écrans existants (formulaires, listes, navigation).
  - Ajout ou modification de composants UI réutilisables.
  - Correction de comportements (validation, messages d’erreur, états de chargement).

- **Intégration API / Backend Laravel**
  - Connexion à de nouveaux endpoints ou adaptation d’appels existants.
  - Gestion des réponses et des erreurs côté mobile.
  - Alignement des modèles côté app avec les structures JSON de l’API.

- **Améliorations UX**
  - Affichage plus clair des statuts (chargement, succès, échec).
  - Affinage des textes, labels, erreurs pour guider l’utilisateur.
  - Petites optimisations de navigation (retours, transitions, feedback visuel).

- **Documentation technique**
  - Rédaction de **docs structurées** sur :
    - l’architecture globale (React Native ↔ Laravel ↔ base de données),
    - les écrans et flux clés,
    - les endpoints API utilisés,
    - les règles métier importantes.
  - Objectif : permettre à d’autres développeurs de rejoindre le projet plus facilement.

## Stack technique

- **Frontend mobile**
  - React Native
  - Écosystème JavaScript
  - Composants UI personnalisés

- **Backend**
  - Framework **Laravel**
  - API REST pour l’app mobile
  - Gestion de l’authentification, des profils, des données métiers

- **Base de données & infra**
  - Base de données relationnelle (MySQL / MariaDB, selon le déploiement)
  - Hébergement serveur classique (stack PHP/Laravel)

- **Outils & workflow**
  - Git / GitHub pour la gestion de versions
  - Tickets / issues pour le suivi des tasks
  - Documentation technique versionnée avec le code

## Rôle & Contribution

Je ne suis **pas le développeur principal** de Div-In, mais j’interviens comme **développeur de soutien** sur :

- l’implémentation de **fonctionnalités ciblées** (mobile + backend),
- la **résolution de bugs** sur les écrans existants,
- l’amélioration de l’**ergonomie** et des retours utilisateur,
- la **rédaction de documentation technique claire et structurée**.

En résumé, mon rôle est de :

- **fiabiliser** des parties de l’app,
- **fluidifier** l’UX à travers des petits ajustements,
- **clarifier** le fonctionnement de l’ensemble par la doc.
