#  Giron de Porsel 2026 – Infrastructure & Outils internes

Dans le cadre de l’organisation du **Giron de Porsel 2026**, j’ai réalisé et maintiens l’ensemble de l’infrastructure numérique de l’événement.  
Cette infrastructure est hébergée sur mon **serveur Dell PowerEdge R730**, sous Linux, avec un **stack Docker complet et isolé**.

L’objectif : fournir des services robustes, sécurisés et accessibles aux différents membres du comité d’organisation.

---

#  Projet 1 – Infrastructure self-hostée (serveur Dell R730)

###  Objectifs
- Centraliser tous les services du Giron sur un seul serveur fiable  
- Sécuriser les accès publics et internes  
- Fournir un environnement stable pour WordPress, Nextcloud et la base de données  

###  Configuration principale
- **OS : Ubuntu Server**  
- **Virtualisation : Docker + docker-compose**  
- **Reverse-proxy : Nginx Proxy Manager ou Traefik**  
- **Gestion certificats HTTPS : Let’s Encrypt / Certbot**  
- **Surveillance : logs + monitoring de conteneurs**  

###  Sécurité & réseau
- Isolation via **networks Docker**
- Règles firewall strictes (ports ouverts uniquement : 80/443)
- Sous-réseau interne pour les services sensibles (DB, Nextcloud)
- Sauvegardes Snapshot + export DB réguliers

#  Projet 2 – Site officiel WordPress

Le site du Giron 2026 est construit sur un stack WordPress moderne, hébergé dans un conteneur dédié.

###  Stack
- **WordPress (Docker)**  
- **Thème Astra Pro**  
- **Éditeur : Elementor Pro**  
- **Plugins : SEO, sécurité, cache, formulaires**  

###  Fonctionnalités du site
- Page d’accueil dynamique  
- Présentation des commissions  
- Actualités & Articles  
- Informations pratiques (billetterie, horaires, animations)  
- Mise en page fidèle à la maquette fournie  
- Performances optimisées (cache + compression + CDN local)  

###  CI/CD simplifié
- Déploiement via `docker-compose pull && docker-compose up -d`  
- Sauvegardes automatiques des volumes WordPress

#  Projet 3 – Nextcloud (Documents & Organisation interne)

Nextcloud sert à centraliser :  
- les documents officiels,  
- les fichiers des commissions,  
- les visuels, maquettes et ressources,  
- les exports comptables,  
- la coordination interne.

###  Fonctionnalités activées
- Partage sécurisé entre membres du comité  
- Gestion des permissions par commission  
- Versioning automatique  
- Stockage + aperçu de médias  
- Collabora Online pour édition collaborative  

###  Stack Nextcloud
- Conteneur Docker Nextcloud  
- Base de données dédiée (MariaDB / PostgreSQL)  
- Redis pour optimiser le cache  
- Monté derrière le reverse-proxy + SSL  

---

#  Projet 4 – Base de données interne

###  Usage
- Stockage des données WordPress  
- Métadonnées Nextcloud  
- Outils internes à venir (ex : inscriptions, bénévoles, sponsoring)  

###  Stack
- MariaDB (Docker)  
- Volumes persistants sur SSD  
- Backups automatisés (cron + dump compressé)  

###  Points forts
- Accès limité au réseau Docker interne  
- Isolation totale vis-à-vis des services publics  
- Récupération rapide en cas de corruption  

---

#  Projet 5 – Reverse Proxy & Réseau

###  Objectif
Permettre à tous les services (site, cloud, outils internes) d’être accessibles via HTTPS, tout en sécurisant les accès.

### Configuration
- Reverse proxy avec :  
  - **Traefik** *ou* **Nginx Proxy Manager**  
- Certificats automatiques via Let’s Encrypt  
- Redirections HTTPS + règles de sécurité  
- HSTS, compression, anti-scan, logs  

### Entrées DNS typiques
- `giron2026.ch` → WordPress  
- `cloud.giron2026.ch` → Nextcloud  
- `admin.giron2026.ch` → outils internes
