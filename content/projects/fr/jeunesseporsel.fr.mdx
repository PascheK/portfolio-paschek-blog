# Aperçu général

La **Jeunesse de Porsel** m’a mandaté pour concevoir un écosystème complet regroupant :

- un **site Next.js** moderne et responsive,
- un **module théâtre** avec portraits et galeries,
- un **module fondue** avec parcours multi-étapes et OTP,
- un **annuaire** des membres & du comité,
- un **centre de contact** sécurisé,
- et un **backend Spring Boot** gérant les événements, inscriptions, emails et codes OTP.

L’ensemble forme une solution unifiée destinée à améliorer la gestion interne de la société et l’expérience des visiteurs.

---

#  1. Frontend – Site Next.js
Le site est la vitrine principale de la Jeunesse de Porsel.  
Il regroupe tous les modules destinés au public : théâtre, fondue, annuaire, contact.

## Fonctionnalités principales

###  Page d’accueil
- Hero visuel
- CTA vers Théâtre, Fondue, Giron 2026
- Présentation de la société

###  Module Théâtre
- Carrousel Embla pour les visuels du spectacle  
- Synopsis & présentation des rôles  
- Portraits des acteurs avec zoom  
- Informations pratiques (horaires, billetterie)

###  Module Fondue – Réservations
- Parcours multi-étapes :
  - choix de la date
  - nombre de personnes
  - données personnelles  
- Validation stricte via **Zod + RHF**
- Vérification par **OTP** (6 chiffres)
- Modification sécurisée d’une réservation
- Emails de confirmation

###  Annuaire de la société
- Comité  
- Membres  
- Données centralisées via store interne

###  Centre de contact
- Formulaire typé (Zod)
- Envoi via API Route Next.js
- Transport d’email avec **Nodemailer**
- Double envoi : administration + utilisateur

---

##  Stack technique frontend

- **Next.js 15** (App Router)
- **React 19 RC**
- **TypeScript**
- **Tailwind CSS**
- **Radix UI**
- **Embla Carousel**
- **React Hook Form + Zod**
- **Nodemailer**
- **Vercel Analytics**

---

##  Rôle & contributions (Frontend)

- Conception UI/UX complète du site  
- Module Théâtre : pages, carrousels, portraits  
- Module Fondue : multi-étapes, OTP, emails  
- Annuaire du comité et des membres  
- Formulaire de contact et API email  
- Animations, responsive design et optimisations

#  2. Backend – API Spring Boot

Le backend centralise toute la logique métier.  
Il communique avec le frontend Next.js via une API REST documentée et sécurisée.

## Fonctionnalités principales

###  Gestion des événements
- Création et listing des événements (fondue, théâtre)
- Définition des capacités de places

###  Gestion des inscriptions
- Création d’une réservation  
- Mise à jour atomique du quota disponible  
- Vérification stricte des doublons (nom + email + événement)  
- Modification sécurisée via OTP

###  Workflow OTP
- Code à 6 chiffres  
- Expiration après 5 minutes  
- Destiné à la validation & modification d’inscriptions  

###  Emails transactionnels
- Confirmation  
- Modification  
- Notification interne  
- Templates FreeMarker personnalisés  
- Envois via JavaMail

###  Observabilité & documentation
- Endpoints **Actuator**
- **OpenAPI / Swagger UI**
- Logs de suivi

---

##  Stack technique backend

- Spring Boot (Web, Validation, JPA, Actuator)  
- Java 17  
- MariaDB + JDBC  
- Lombok  
- Spring Mail  
- FreeMarker (templates emails)  
- OpenAPI / Swagger UI  
- Maven Spring Boot Plugin  

---

##  Rôle & contributions (Backend)

- Conception du modèle de données  
- Création de l’API REST complète  
- Règles métier : capacité, doublons, transactions  
- Implémentation complète du workflow OTP  
- Configuration des emails (JavaMail + FreeMarker)  
- Documentation OpenAPI & supervision Actuator  
- Optimisation MariaDB et indexation
