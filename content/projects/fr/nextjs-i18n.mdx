Construire un site multilingue dans Next.js (App Router) implique un équilibre entre DX, performance et clarté. Ce projet met en œuvre :

### Fonctionnalités clés

- Segment de langue (`/[lang]`) avec params statiques pour `en` et `fr`.
- Chargeur de dictionnaires via fichiers JSON par langue.
- Contenu MDX organisé par langue (`/content/blog/{lang}` et `/content/projects/{lang}`).
- Métadonnées localisées + schéma JSON-LD incluant les URLs canoniques.
- Théming (clair/sombre) avec design tokens OKLCH.
- Transitions de page animées et composants de révélation au scroll.
- Pages de détail projet & blog partageant une même structure visuelle.

### Logique de filtrage

La page projets applique une logique dérivée sans effets de bord :

```ts
const filteredProjects = useMemo(
  () =>
    selected === "__all__"
      ? all
      : all.filter((p) => p.categories.includes(selected)),
  [all, selected]
);
```

La liste complète reste toujours la source de vérité.

### Page À propos

Remplace une ancienne page Photos par un mini-CV structuré (expériences, compétences, éducation, lien CV) — tout le texte vit dans les dictionnaires pour simplifier la traduction.
